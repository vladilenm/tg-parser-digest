# SEED-002: v5.0 — Дашборды + Bitsab integration

Web-дашборд с историей сводок, фильтрами по 5 направлениям и графиками + интеграция Bitsab (или аналога) как источника ценовых данных по нефтепродуктам, привязанных к новостным событиям.

## Why This Matters

Четыре драйвера, каждый достаточен сам по себе:

1. **Расширение договора (Этап 3+).** Естественное продолжение работ с Заказчиком после Этапов 1 (v3.0, 275к) и 2 (v4.0). Дашборды + цены — материал для нового этапа договора №2020 или отдельного контракта. Без визуализации и ценового контекста дальнейшая работа упирается в потолок Telegram-формата.

2. **Цены — сравнение фактов и рынка.** Bitsab даёт реальные цены на нефтепродукты (бункерное топливо, масла, керосин, битум — те же 5 направлений, что и в дайджесте). Новость + цена = полный контекст: «Роснефть снижает экспорт битума» становится осмысленным, когда видна реакция цены. Это именно то, что Заказчик использует в принятии решений.

3. **Web-интерфейс вместо Telegram.** Telegram-вывод упирается в лимиты: 4096 символов, нет фильтров, нет истории, нет поиска, нет графиков. Дашборд снимает все эти ограничения: фильтры по направлениям/компаниям/датам, история за недели/месяцы, графики цен поверх событий, ссылки на verified-источники из v4.0.

4. **Платные клиенты / масштаб.** Дайджест + verification (v4.0) + ценовой контекст + дашборд — это уже полноценный продукт, который можно предлагать другим игрокам нефтегаза (трейдеры, брокеры, аналитические службы НПЗ) помимо текущего Заказчика. Стратегический мост к коммерциализации.

## Scope Estimate

**Medium — фаза или две.**

MVP-уровень дашборда + цен:

- **Phase A (Bitsab integration):** подключение Bitsab API (или ближайшего аналога — Argus, S&P Platts при наличии ключа; иначе бесплатные публичные ленты). Цены подтягиваются на каждом прогоне daemon, привязываются к событиям из дайджеста по направлению (бункер/масла/керосин/нефтехимия/битум). Реализовать как новую секцию в существующем pipeline (см. [src/pipeline.ts](../../src/pipeline.ts) после v3.0 рефакторинга).
- **Phase B (статический HTML-дашборд):** генерация статического `dashboard.html` при каждом прогоне поверх `data/output/*.md` и `data/raw/*.json` за последние N дней. Фильтры по направлениям, графики цен (Chart.js без бэкенда), ссылки на исходные посты и на verified-источники. Хостится как файл на VPS (nginx static) или GitHub Pages с приватным доступом.

Что НЕ входит в Medium и должно быть Out of Scope для v5.0:

- Полноценный SPA-фронтенд (React/Vue) — статичной HTML-генерации достаточно для MVP
- Серверная база данных (Postgres/SQLite) — `data/` директория продолжает быть source of truth
- Аутентификация / multi-tenancy — один оператор, basic auth на nginx
- Real-time через WebSocket — обновление раз в сутки достаточно (привязка к расписанию daemon)
- Платные ценовые источники (Argus, Platts) — только при наличии ключа от Заказчика; иначе Bitsab или бесплатные

Если объём растёт за рамки 2 фаз (например, требование БД + auth + SPA одновременно) — переоценить как Large и разделить v5.0/v5.1.
