Replace SYSTEM_PROMPT and summarize() function with:

**CLASSIFY_SYSTEM_PROMPT** (new):
- Role: classifier for Russian oil/gas/petrochemistry posts
- Input: JSON {posts: [{url, text}]}
- Task: for each post determine category (bunker|oil|kerosene|petrochem|bitumen|null) and mentions (rosneft|lukoil|gazprom — array, can be empty)
- Output: JSON {classifications: [{url, category, mentions}]}
- Rules: category=null if post does not fit any of 5 categories; mentions[] lists companies explicitly named in text; be conservative — only assign category when clearly relevant

**SUMMARIZE_CATEGORY_PROMPT** (new):
- Role: extractive editor for a single category of Russian oil/gas posts
- Input: JSON {category: string, posts: [{url, channelUsername, text}]}
- Task: for each post write summary (1-2 sentences, up to 250 chars Russian) + extract keyQuote (MUST be verbatim substring of text). If multiple posts describe the same event/story — merge into one item picking the most informative keyQuote.
- Output: JSON {items: [{summary, keyQuote, url, channel, mentions}]}
- Rules: keyQuote must be exact substring of text (server verification follows). mentions[] = companies from {rosneft, lukoil, gazprom} found in text. Return ALL relevant items — no count limit.

**classifyPosts(client, posts, model)** async function:
- Sends 1 LLM call with CLASSIFY_SYSTEM_PROMPT
- User message: JSON.stringify({posts: posts.map(p => ({url: p.url, text: p.text}))})
- Parses response with ClassificationResponseSchema.safeParse
- On schema fail: retry once with correction system message (same pattern as current code)
- On second fail: throw
- Returns parsed classifications array
- log.info before/after call with timing

**summarizeCategory(client, category, posts, model)** async function:
- Sends 1 LLM call with SUMMARIZE_CATEGORY_PROMPT
- User message: JSON.stringify({category, posts: posts.map(p => ({url: p.url, channelUsername: p.channelUsername, text: p.text}))})
- Parses response with CategoryItemsResponseSchema.safeParse
- On schema fail: retry once
- On second fail: log.warn(`[summarize] category ${category} failed after retry — skipping`) and return empty array (ANTIFRAGILE — do not throw)
- Returns array of CategoryItemSchema items
- log.info before/after call with timing

**New summarize() orchestrator** (same export signature):
```
export async function summarize(posts: Post[], channelStats?: ChannelStats): Promise<{ html: string; postsDropped: number }>
```

Implementation:
1. Setup: read env vars (DEEPSEEK_API_KEY, DEEPSEEK_BASE_URL, DEEPSEEK_MODEL), create OpenAI client with timeout: 120_000, maxRetries: 1
2. Pass 1: const classifications = await classifyPosts(client, posts, model)
3. Bucket posts: create Map<string, Post[]> for each non-null category + "mentions" bucket. Logic:
   - category != null → bucket by category
   - category == null AND mentions.length > 0 → "mentions" bucket
   - category == null AND mentions.length == 0 → dropped (irrelevant)
   - Log: log.info(`[summarize] classification: ${JSON.stringify(Object.fromEntries([...buckets].map(([k,v])=>[k,v.length])))}`)
4. Pass 2: Run Promise.allSettled on all non-empty buckets. For each bucket call summarizeCategory(client, category, posts_in_bucket, model).
5. Assemble DigestJson: For each settled result, if fulfilled — inject category field into each item (the LLM returns items without category; orchestrator sets it). Category for "mentions" bucket items = null. If rejected — log.warn, treat as empty.
6. Build DigestJson object: {generatedAt: new Date().toISOString(), bunker: [...], oil: [...], kerosene: [...], petrochem: [...], bitumen: [...], mentions: [...]}
7. verifyExtractiveness(digest, posts) — unchanged
8. renderHtml(digest, posts, channelStats) — unchanged
9. Return {html, postsDropped: droppedCount}

Key details:
- The OpenAI client instance is shared across all calls (connection pooling)
- Use response_format: {type: "json_object"} for both passes
- Preserve all log.info calls for observability
- Total LLM calls: 1 (classify) + N (summarize per non-empty bucket, max 6)
