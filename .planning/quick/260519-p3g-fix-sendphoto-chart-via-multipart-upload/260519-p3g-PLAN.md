<objective>
Fix production bug: Telegram API возвращает `400 Bad Request: failed to get HTTP URL content` при попытке скачать quickchart short URL через `sendPhoto(photo: url)`. Это известная проблема `sendPhoto by URL` — race condition / TG-кеш / шейпинг сторонних доменов.

Решение: переключиться с `sendPhoto by URL` на multipart upload. Скачиваем PNG локально (quickchart возвращает PNG bytes напрямую при POST на `/chart`), потом постим в TG как `multipart/form-data` с полем `photo` = Blob(image/png).

Purpose: Восстановить доставку визуального чарта top-10 НПЗ в /summarize. На прошлом запуске narrative дошёл, но чарт упал на shot URL → photo missing.

Output:
- Переписанный `src/upload/chart.ts` — endpoint `/chart` (PNG bytes) вместо `/chart/create` (JSON {url}).
- Переписанный `handleSummarizeCommand` в `src/bot.ts` — multipart fetch без `tgFetch`.
- Переписанные тесты под новый contract — arrayBuffer mock + FormData проверки.
- Старые 8 тестов /summarize остаются зелёные (default mock generateChartPng→null).
</objective>

<implementation-notes>
**Quickchart endpoint contract:**
- POST `https://quickchart.io/chart` — body тот же JSON ({chart, backgroundColor, width, height, format: "png"}), но response — это сам PNG (бинарь, `Content-Type: image/png`), НЕ JSON.
- Берём `await res.arrayBuffer()` → `new Uint8Array(buf)`.
- На !res.ok → throw (как раньше).
- Quickchart `/chart` рендерит синхронно, поэтому таймаут увеличиваем до 15s (DEFAULT_TIMEOUT_MS = 15_000).

**Telegram sendPhoto multipart:**
- URL: `https://api.telegram.org/bot${token}/sendPhoto` (тот же, что использует tgFetch).
- НЕ передавать никаких headers — `fetch` сам выставит `Content-Type: multipart/form-data; boundary=...` когда `body` это `FormData`.
- Поля FormData:
  - `chat_id` = String(chatId)
  - `photo` = `new Blob([png], { type: "image/png" })` с третьим аргументом filename `"chart.png"` (нужен Telegram'у для определения формата).
  - `caption` = "Δ цены и объёмы по НПЗ (top-10)"
- На !res.ok → log.warn, НЕ throw (caller-level try/catch уже есть для безопасности, но soft-fail чище).

**Tests rewrite:**
- `upload-chart.test.ts` — секции `fetchQuickChartUrl` и `generateChartUrl` переименовываем в `fetchQuickChartPng` / `generateChartPng`, mock response переводим с `json: async () => ({success: true, url: ...})` на `arrayBuffer: async () => new Uint8Array([0x89, 0x50, 0x4E, 0x47]).buffer`. Проверка URL — `calledUrl).toContain("quickchart.io/chart")` (а НЕ `/chart/create`).
- `buildChartConfig` тесты — НЕ ТРОГАЕМ (он pure builder, контракт неизменен).

Default mock в beforeEach: `mockedGenerateChartPng.mockResolvedValue(null)` — старые тесты не упадут.
- Тест "tgFetch sendPhoto returns 400" → переписать в "sendPhoto multipart returns 400": fetch mock возвращает {ok: false, status: 400}, handler не падает, fetch вызван 1 раз с FormData body.
