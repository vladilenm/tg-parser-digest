<objective>
Сделать конфиг и state-файлы анти-хрупкими к пересборке Docker-контейнера: перенести `channels.json`, `websites.json`, `hash-cache.json`, `web-posts-*.json`, `data/raw`, `data/output`, `data/run-*.log` в persistent volume `/app/data/{config,state,raw,output,logs,backups}`. Реализовать seed-логику (на первом старте копировать дефолты из `/app/seed/` → `/app/data/config/`), daily backup в закрытый Telegram-канал в 03:15 MSK и pre-deploy snapshot на VDS через `deploy.sh`.

Purpose: Push в `main` пересоздаёт контейнер на Timeweb Cloud Apps и затирает мутабельные данные (каналы, добавленные через `/add_channel`, накопленный 14-дневный hash-cache, архивы прогонов). Эти данные критичны: список каналов — единственный источник правды для пайплайна, hash-cache защищает от повторов, архивы — acceptance-доказательство для Заказчика.

Output: Один контейнер с одним volume `/app/data`, где переживают пересборку все мутабельные файлы. Daily backup `.tgz` падает в Telegram-канал каждый день в 03:15 MSK. Pre-deploy snapshot создаётся на VDS перед каждым `git pull` в `/opt/backups/pre-deploy-${TS}.tgz` (хранятся 5 последних). Vendor-lock-in нет — переезд на VDS / другой провайдер сводится к копированию `/app/data/`.

Spec: `docs/db-deploy.md` — план полностью описан там, согласован с пользователем (Variant A, volume-only, без новой БД). Этот PLAN.md — атомарная декомпозиция спеки в 5 коммитов.
