<objective>
Форсировать HTTP/1.1 в `fetchSite` через кастомный `undici.Agent` (`allowH2: false`) и расширить заголовки до браузерных, чтобы убрать массовые `fetch failed` после quick-260508-cha (timeout=30s).

Purpose: Native Node fetch (undici) договаривается на HTTP/2 ALPN, и для серверов с Tengine (`neftegaz.ru`), Bitrix-стеком и корпоративных WAF либо роняет h2-frames, либо TLS-fingerprint не совпадает. `curl` с тем же UA получает HTTP/2 200, т.е. проблема в h2-pipeline undici, не в headers/UA. Стандартный фикс — кастомный dispatcher с `allowH2: false`. Доп. browser-like headers закрывают edge-cases на сайтах, фильтрующих по `sec-fetch-*`/`accept-language`.

Output: один отредактированный `src/web-scraper.ts` (правки в одной функции `fetchSite` + module-level singleton), без новых deps, без новых тестов, без правок других файлов.

```ts
// quick-260508-cy1: undici h1-only dispatcher — обходит HTTP/2 frame-parsing и TLS-fingerprint
// проблемы на сайтах с Tengine (neftegaz.ru), Bitrix-стеком и корпоративных WAF.
// Без него native fetch падает с generic "fetch failed" после успешного TCP/TLS handshake.
// curl --http1.1 с тем же UA проходит на тех же URL — корневая причина именно в h2-pipeline undici.
const httpDispatcher = new Agent({
  allowH2: false,
  connect: { timeout: 10_000 },
});
```

ПРАВКА 3 — в единственном `fetch(currentUrl, { ... })` вызове внутри `fetchSite` (текущие строки 74-79) расширить headers и добавить `dispatcher`. Объект options целиком становится:
```ts
const res = await fetch(currentUrl, {
  method: "GET",
  headers: {
    "user-agent": userAgent,
    "accept": "text/html,application/xhtml+xml,application/xml;q=0.9,*/*;q=0.8",
    "accept-language": "ru-RU,ru;q=0.9,en;q=0.8",
    "sec-fetch-dest": "document",
    "sec-fetch-mode": "navigate",
    "sec-fetch-site": "none",
    "sec-fetch-user": "?1",
    "upgrade-insecure-requests": "1",
```

Cast `as RequestInit & { dispatcher?: unknown }` нужен потому, что DOM-овый `RequestInit` не знает про undici-расширение `dispatcher`. Это самый дешёвый путь — без модификации tsconfig и без `@ts-expect-error` (который рискует протухнуть, если TS подтянет тип).
