# tg-parser-demo — как это работает и как настроить

Документ для оператора: объяснение системы простыми словами, защита от бана, пошаговая первичная настройка и правила работы с `channels.yaml`.

---

## Как работает система — простыми словами

Это один TypeScript-скрипт, который запускается командой `npm start` и делает 3 вещи по цепочке:

### 1. Читает каналы (MTProto, user-аккаунт)
Файл: [src/run.ts:39-65](../src/run.ts#L39-L65)

Парсит [channels.yaml](../channels.yaml), подключается к Telegram через GramJS под твоим личным аккаунтом (не ботом — бот не может читать чужие каналы) и по очереди обходит список. Для каждого канала берёт последние 50 сообщений за 24 часа — [src/telegram.ts:61-101](../src/telegram.ts#L61-L101).

### 2. Отдаёт всё в DeepSeek одним батчем
Файл: [src/summarize.ts:192-232](../src/summarize.ts#L192-L232)

Отправляет все посты одним JSON-запросом к DeepSeek с жёстким system-prompt: «пиши только по фактам из text, keyQuote должен быть дословной подстрокой». Получает JSON с 3–6 темами и до 15 новостей.

### 3. Верификация + отправка в канал (Bot API)
Файлы: [src/summarize.ts:101-143](../src/summarize.ts#L101-L143), [src/deliver.ts:58-92](../src/deliver.ts#L58-L92)

Критический шаг: каждую цитату от LLM прогоняет через `post.text.includes(keyQuote)`. Если дословного совпадения нет — выкидывает. Потом собирает HTML и отправляет ботом в твой приватный канал (режет на части по 4000 символов).

---

## Что самое важное в коде

- **[src/summarize.ts:118-127](../src/summarize.ts#L118-L127)** — серверная проверка дословности цитат. Главный барьер против галлюцинаций LLM. Если отсюда что-то удалить — смысл проекта теряется.
- **[src/telegram.ts:10-16](../src/telegram.ts#L10-L16)** — «правдоподобная идентичность» (Desktop/Windows 11/RU). Anti-ban маскировка.
- **[src/telegram.ts:127-139](../src/telegram.ts#L127-L139)** — обработчик `FloodWaitError` с одним retry. Если второй раз подряд — прогон обрывается.
- **[src/run.ts:59-61](../src/run.ts#L59-L61)** — задержка между каналами с jitter. Вторая линия anti-ban защиты.

## Защита от бана/спама

В коде есть **три защиты**, но **одна критическая дисциплина на тебе**:

### Что защищает код

1. **Идентичность клиента** — [src/telegram.ts:10](../src/telegram.ts#L10): скрипт представляется нормальным Telegram Desktop, а не «подозрительным ботом».
2. **Задержка между каналами** — `CHANNEL_DELAY_MS=1000` + случайный jitter 0–500мс. Не дёргает API пачкой.
3. **FloodWait handling** — если Telegram сказал «подожди N секунд», скрипт спит `N+2` секунды и делает **один** retry. Второй FloodWait подряд → `exit 1`, больше не лезем.

### Чего в коде НЕТ (и что делаешь ты)

- **Нет rate-limit между запусками.** Правило из [README.md](../README.md) и [CLAUDE.md](../CLAUDE.md): **не чаще одного прогона в 10–15 минут**. Нарушишь — получишь FloodWait, в худшем случае временный бан user-аккаунта (не бота — user-аккаунта, того самого, где лежит `TG_SESSION`).
- **Не реже, чем раз в сутки.** Окно чтения 24ч. Частые запуски = одни и те же посты, зря грузишь API.
- **Риск бана при FloodWait:** если ты проигнорируешь exit 1 и запустишь снова — Telegram может расценить как abuse. Если скрипт упал на втором FloodWait — жди 30–60 минут.

**Вывод:** код защищён на уровне «делаю один запрос корректно». От массовости защищает только твоя дисциплина запуска.

## Пошаговый план первичной настройки

### Шаг 0. Установка

```bash
npm install
cp .env.example .env
```

### Шаг 1. Заполнить `.env` — где брать каждую переменную

| Переменная | Откуда взять |
|---|---|
| `TG_API_ID` и `TG_API_HASH` | Зайди на **https://my.telegram.org** под своим номером → **API development tools** → **Create application** (название любое, например `tg-parser-demo`, платформу ставь Desktop). После создания тебе покажут `api_id` (число) и `api_hash` (хэш). Копируй в `.env`. |
| `TG_SESSION` | **Не заполняй руками.** Генерируется на Шаге 2. |
| `TG_BOT_TOKEN` | Открой **@BotFather** в Telegram → `/newbot` → придумай имя и username боту. BotFather вернёт токен вида `123456:ABC-...`. Копируй в `.env`. |
| `TG_CHANNEL_ID` | 1) Создай новый **приватный** канал в Telegram (любой клиент → New Channel → Private). 2) Добавь туда своего бота как **администратора** с правом **Post Messages**. 3) Перешли любое сообщение из этого канала боту **@username_to_id_bot** — он вернёт ID формата `-100xxxxxxxxxx`. Копируй в `.env` целиком, вместе с минусом. |
| `DEEPSEEK_API_KEY` | Регистрация на **https://platform.deepseek.com** → раздел **API keys** → Create new key → пополни баланс (несколько долларов хватит надолго). Копируй в `.env`. |
| `DEEPSEEK_MODEL` | Оставь `deepseek-chat`. |
| `DEEPSEEK_BASE_URL` | Оставь `https://api.deepseek.com`. |
| `FETCH_WINDOW_HOURS` | Оставь `24`. |
| `MAX_MESSAGES_PER_CHANNEL` | Оставь `50`. |
| `CHANNEL_DELAY_MS` | Оставь `1000` (меньше — риск FloodWait). |
| `LOG_LEVEL` | `info` для обычной работы, `debug` для отладки. |

### Шаг 2. Сгенерировать `TG_SESSION`

```bash
npm run login
```

Скрипт [scripts/login.ts](../scripts/login.ts) спросит:

1. **Телефон** в международном формате (например, `+79991234567`) — тот самый номер, под которым ты подписан на нужные каналы.
2. **Код из Telegram** — придёт в официальное приложение Telegram.
3. **2FA-пароль** — если включён облачный пароль. Если нет — Enter.

После успеха он напечатает длинную строку `StringSession`. **Скопируй её в `.env` как `TG_SESSION`**. Это полный доступ к твоему аккаунту — никуда не выкладывай.

### Шаг 3. Подписаться на каналы

Открой Telegram под тем же номером, под которым делал `npm run login`, и **подпишись вручную** на все каналы из [channels.yaml](../channels.yaml). Если не подписан — скрипт выкинет `ChannelPrivateError` и канал пропустится ([src/telegram.ts:114-124](../src/telegram.ts#L114-L124)).

### Шаг 4. Первый прогон

```bash
npm start
```

Через ~30–60 секунд в твой приватный канал прилетит HTML-дайджест.

## Как правильно заполнять `channels.yaml`

Формат предельно простой — [channels.yaml](../channels.yaml):

```yaml
channels:
  - username: "neftegazru"
    priority: 1
  - username: "oilcapital"
    priority: 3
```

### Правила

1. **Только `username` публичного канала** — то, что идёт после `t.me/` или `@` в Telegram. Для канала `https://t.me/neftegazru` username = `neftegazru`.
2. **Без `@`, без ссылок, без `t.me/`** — только чистый username в кавычках.
3. **Только публичные каналы.** Приватные по invite-link (`t.me/+AbC123...`) **не поддерживаются** — см. [README.md](../README.md).
4. **Поле `priority`** — зарезервировано, в MVP не используется. Можно ставить любое число или вообще опустить (валидатор [src/run.ts:20-37](../src/run.ts#L20-L37) требует только `username`).
5. **10–15 каналов** — рекомендованный объём. Больше — риск FloodWait, больше расход токенов DeepSeek.
6. **Ты должен быть подписан** на каждый канал тем же аккаунтом, что в `TG_SESSION`. Иначе пропуск с warning.

### Как добавить свой канал

1. Найди его в Telegram, убедись что он публичный (есть `@username`).
2. Подпишись своим аккаунтом.
3. Допиши в `channels.yaml`:
   ```yaml
     - username: "my_new_channel"
       priority: 2
   ```
4. Сохрани, запусти `npm start`.

### Частая ошибка

- ❌ `username: "@neftegazru"` — с собакой, не сработает
- ❌ `username: "https://t.me/neftegazru"` — со ссылкой, не сработает
- ✅ `username: "neftegazru"` — правильно

## Коротко итог

Один скрипт, user-аккаунт читает, LLM сокращает, бот постит. Защита от бана = идентичность + задержки + FloodWait-retry в коде, плюс **твоя дисциплина не запускать чаще 1 раза в 10–15 минут**. Для настройки нужно 5 секретов (2 от my.telegram.org, токен от BotFather, ID приватного канала, ключ DeepSeek) + одна сессионная строка через `npm run login`.
